Installation guide

This document is a guide for installing Arch Linux using the live system booted from an installation medium made from an official installation image. The installation medium provides an up-to-date Arch Linux system with a set of command-line tools, and can be used to install packages, set up the system, and perform the initial configuration.

Before installing, it is advised to view the FAQ. For conventions used in this document, see Help:Reading.

Pre-installation

Note: If you are installing from a wireless connection, see Wireless network configuration.
Warning: The installation images do not support Secure Boot. You will need to disable Secure Boot to boot the installation medium. If desired, Secure Boot can be set up after completing the installation.
Tip: For alternative ways of installation, see Category:Installation process.

Verify the boot mode

To verify the boot mode, check the UEFI bitness:

# cat /sys/firmware/efi/fw_platform_size

If the command returns 64, then system is booted in UEFI mode and has a 64-bit x64 UEFI. If the command returns 32, then system is booted in UEFI mode and has a 32-bit IA32 UEFI; while this is supported, it will limit the boot loader choice to EFISTUB, systemd-boot and GRUB. If the file does not exist, the system may be booted in BIOS (or CSM) mode.

Connect to the internet

To set up a network connection in the live environment, go through the following steps:

  • Ethernet—plug in the cable.
  • Wi-Fi—authenticate to the wireless network using iwctl.
  • Mobile broadband modem—connect to the mobile network with the mmcli utility.

To verify the connection, use ping:

# ping archlinux.org
Network connection types
Connection type Service/Tool Configuration Notes
Ethernet (wired) dhcpcd Automatic on boot Plug and play
Wi-Fi (wireless) iwd, iwctl Manual authentication WPA2/WPA3 support
Mobile broadband ModemManager Manual setup 3G/4G/5G modems
Bridge/Bond netctl Advanced configuration Multiple interfaces

Update the system clock

Use timedatectl to ensure the system clock is accurate:

# timedatectl set-ntp true

To check the service status, use timedatectl status.

Installation

Partition the disks

When recognized by the live system, disks are assigned to a block device such as /dev/sda, /dev/nvme0n1 or /dev/mmcblk0. To identify these devices, use lsblk or fdisk.

#!/bin/bash
# List all block devices
lsblk

# Alternative: use fdisk to list disks
fdisk -l

# Create partitions with fdisk
fdisk /dev/sda

# Create partitions with cfdisk (more user-friendly)
cfdisk /dev/sda

# For UEFI systems, create:
# 1. EFI System Partition (512MB, type EF00)
# 2. Root partition (remaining space, type 8300)

# For BIOS systems, create:
# 1. Boot partition (512MB, bootable)
# 2. Root partition (remaining space)

Results from lsblk may resemble the following:

NAME        MAJ:MIN RM   SIZE RO TYPE MOUNTPOINTS
sda           8:0    0 298.1G  0 disk 
├─sda1        8:1    0   512M  0 part /boot/efi
├─sda2        8:2    0   297G  0 part /
└─sda3        8:3    0   600M  0 part [SWAP]

Format the partitions

Once the partitions have been created, each newly created partition must be formatted with an appropriate file system.

For example, to create an ext4 file system on /dev/root_partition, run:

# mkfs.ext4 /dev/root_partition

If you created a partition for swap, initialize it with mkswap:

# mkswap /dev/swap_partition
Note: For stacked block devices, replace /dev/*_partition with the appropriate block device path.

If you created an EFI system partition, format it to FAT32 using mkfs.fat:

# mkfs.fat -F 32 /dev/efi_system_partition

Configure the system

Fstab

Generate an fstab file (use -U or -L to define by UUID or labels, respectively):

# genfstab -U /mnt >> /mnt/etc/fstab

Check the resulting /mnt/etc/fstab file, and edit it in case of errors.

Chroot

Change root into the new system:

# arch-chroot /mnt

Time zone

Set the time zone:

# ln -sf /usr/share/zoneinfo/Region/City /etc/localtime

Run hwclock to generate /etc/adjtime:

# hwclock --systohc

Reboot

Exit the chroot environment by typing exit or pressing Ctrl+d.

Optionally manually unmount all the partitions with umount -R /mnt: this allows noticing any "busy" partitions, and finding the cause with fuser.

Finally, restart the machine by typing reboot: any partitions still mounted will be automatically unmounted by systemd. Remember to remove the installation medium and then login into the new system with the root account.
